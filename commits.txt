# Exemplos rápidos
* feat: nova funcionalidade
* fix(scope): correção de bug no escopo
* feat!: mudança que quebra a compatibilidade / feat(scope)!: reestruturação da API
* chore(deps): atualização de dependências

# Tipos de commits
* build: Mudanças que afetam o sistema de build ou dependências externas (exemplos de escopos: gulp, broccoli, npm)
* conf: Mudanças em arquivos e scripts de configuração
* chore: Mudanças que não alteram o código-fonte ou testes, como alterações no processo de build, ferramentas auxiliares, bibliotecas
* docs: Apenas alterações na documentação
* feat: Uma nova funcionalidade
* fix: Uma correção de bug
* perf: Uma alteração de código que melhora o desempenho
* refactor: Uma alteração de código que não corrige um bug nem adiciona uma funcionalidade
* revert: Reverter algo
* style: Mudanças que não afetam o significado do código (espaços em branco, formatação, ponto e vírgula ausente, etc)
* test: Adição de testes ausentes ou correção de testes existentes
